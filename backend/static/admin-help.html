Infra-Vision / Guia do Admin

Voltar ao painel

Adicionar um novo agente

  1. No servidor Linux de destino, copie o arquivo install_agent.sh
  2. Crie o arquivo env na mesma pasta com o token: 
    BACKEND_URL=http://SEU_SERVIDOR:8000
AGENT_TOKEN=seu_token_aqui
  3. Execute: sudo bash install_agent.sh
  4. O agente aparecera automaticamente no painel em ate 5 minutos
  5. Edite o agente no painel para adicionar localizacao e nome amigavel

Criar um cliente

  1. No painel admin, va ate a secao Clientes
  2. Clique em "+ Novo Cliente"
  3. Preencha: username, senha, e os hostnames que o cliente pode ver (separados por virgula)
  4. O cliente acessa o portal em /client/login com as credenciais criadas

Acoes em lote (bulk)

Use os checkboxes na tabela de agentes para selecionar multiplos hosts. Opcoes:

  • Sel. offline — Seleciona automaticamente todos os agentes com status offline
  • Restart / Update / Decommission — Executa o comando em todos os selecionados

Diagnostico remoto

No menu de acoes (⋯) de cada agente:

  • Bind9 Validacao — Executa named-checkconf e named-checkzone
  • DIG Teste — Resolve dominios de teste e mostra latencia
  • DNS Trace — Trace completo da resolucao de um dominio especifico

Troubleshooting

Agente nao aparece no painel

  • Verifique se o servico esta rodando: systemctl status dns-agent
  • Verifique os logs: journalctl -u dns-agent -f
  • Confirme que BACKEND_URL e AGENT_TOKEN estao corretos no /etc/dns-agent/env
  • Teste conectividade: curl -s http://SEU_SERVIDOR:8000/health

Comando nao executa

  • O agente faz polling a cada 60 segundos — aguarde ate 1 minuto
  • Se NATS estiver conectado, a entrega e quase instantanea
  • Verifique o historico de comandos no painel — status "expired" indica que o agente nao buscou a tempo

API

A documentacao completa da API esta disponivel em /docs (Swagger/OpenAPI).

Exemplos com curl

# Listar agentes
curl -H "Authorization: Bearer $TOKEN" http://localhost:8000/api/v1/agents

# Enviar comando
curl -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"hostname":"dns01","command":"restart","issued_by":"admin"}' \
  http://localhost:8000/api/v1/commands

# Health check
curl http://localhost:8000/health
